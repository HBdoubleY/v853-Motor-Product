# 存储进度条阈值变色方案

## 目标

- 当存储使用率 `> 80` 时，进度条前景从默认绿色切换为红色。
- 在白天/黑夜主题均生效，不写死颜色值，统一使用主题宏。

## 变更文件

- [/home/hyby/Desktop/26-3-11-v853/lvgl-gui/ui/generated/MediaLibrary/storageLibrary.c](/home/hyby/Desktop/26-3-11-v853/lvgl-gui/ui/generated/MediaLibrary/storageLibrary.c)

## 实施步骤

- 在 `storageLibrary.c` 增加一个内部小函数（例如 `update_progress_bar_color_by_percentage(int percentage)`）：
  - `percentage > 80`：`LV_PART_INDICATOR` 使用 `TM_ERROR`
  - 否则使用 `TM_SUCCESS`
- 在 `update_storage_display()` 中，计算出 `percentage` 后、`lv_bar_set_value(...)` 附近调用该函数，确保每次进度刷新时颜色同步更新。
- 在“未挂载/无效数据”分支（`total_mb <= 0 || used_mb < 0`）将进度条置 `0` 时，恢复为默认 `TM_SUCCESS`，避免残留红色状态。
- 在 `create_ui_components()` 保留初始化默认绿色（或复用同一函数传 `0`），保证首次进入页面样式一致。

## 主题兼容性说明

- 使用 `TM_ERROR` / `TM_SUCCESS` 而非固定 `lv_color_hex(...)`，颜色会跟随 `theme_manager_get_colors()`，因此白天/黑夜都会取各自主题的红色和绿色。
- 即使主题切换后，后续 `update_storage_display()` 周期刷新会重新套用当前主题对应颜色。

## 验证点

- 使用率 `79%`：前景绿色。
- 使用率 `81%`：前景红色。
- 切换主题后（白天↔黑夜）重复 79/81 场景，颜色在两套主题下都正确。
